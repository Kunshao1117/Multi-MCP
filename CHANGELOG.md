# 更新紀錄

## v0.8.2 — 2026-05-04

### 新增功能

- **GitNexus 程式碼智慧代理整合**：成功將 GitNexus MCP 伺服器接入閘道器，並解決檔案鎖定問題。透過本地靜態路徑執行，提供專案結構掃描、執行流程追蹤與 360 度符號關聯分析能力。

### 文件更新

- **全端技術文件重建**：基於 GitNexus 產出的深度分析圖譜，建立完整、專業級別的 `README.md`，涵蓋架構解析、設定檔說明與全方位的開發指南。

## v0.8.1 — 2026-04-10

### 架構重構與升級

- **Supabase MCP 防彈架構重構**：淘汰不穩定的本地 NPM 互動式命令列架構 (`@supabase/mcp-server-supabase`)，全面改用 Gemini 原生支援的 Remote SSE 遠端伺服器與 OAuth 認證方案，徹底杜絕 Telemetry 與終端機環境變數引發的系統假死問題，並清除殘留廢棄設定檔。
- **Antigravity 框架升級**：自動執行升級機制，同步最新版 `.agents` 工作流程、安全規則與系統腳本庫。


## v0.8.0 — 2026-04-03

### 新增功能

- **Trunk 程式碼品質平台整合**：新增 Trunk MCP Server（HTTP 傳輸模式），AI 代理人可直接呼叫 Trunk 的測試框架偵測、不穩定測試修復與 CI 測試上傳設定等功能。整合採直連 Gemini IDE 全域設定方案（`serverURL` 欄位），繞過本地 Gateway，無需修改閘道器核心。

### 框架與工具更新

- **Antigravity 框架升級至 v5.5.4**：同步最新版框架規則、全套工作流程（00–12）與技能文件，涵蓋跨語系語意守衛優化、安全性合約強化、語意推理深度更新等。
- **`.gitignore` 格式修正**：移除多餘空行，確保 Git 正確識別 `%SystemDrive%` 排除規則。
- **記憶管理設定停用**：`cartridge-system.json` 改為停用狀態（`.disabled` 後綴），由 Gemini IDE 全域管理取代直接 MCP 掛載方式。
- **系統記憶同步**：更新三張模組記憶（系統層、閘道核心、CLI 主控台），重置過期索引，記錄 Trunk MCP 架構決策與 HTTP 傳輸欄位名差異教訓。

## v0.6.0 — 2026-03-31

### 新增功能

- **工具轉發路徑智慧填充**：Gateway 在轉發下游工具呼叫前，自動驗證 AI 填寫的目標專案路徑是否有效（以 `.agents` 目錄存在為判斷依據）。若路徑錯誤則自動修正為工作目錄，若未填寫則自動注入。徹底解決多專案場景中 AI 誤填路徑導致下游工具找不到記憶卡等資源的問題。

### 問題修復

- **工作目錄設定變數遮蔽修正**：修復 `set_workspace` 功能中區域變數名稱與頂部匯入模組同名的遮蔽問題，避免潛在的路徑處理異常。

### 框架與工具更新

- **Antigravity 框架升級至 v4.1.0**：同步最新版框架規則、工作流程與技能文件。

### 測試

- 新增 4 個自動化測試案例（路徑正確不修正、路徑錯誤自動修正、未填自動注入、無工作目錄不填充），全站測試 31 個全數通過。

## v0.5.0 — 2026-03-28

### 新增功能

- **工具呼叫工作目錄強制聲明**：`gateway__call_tool` 新增必填的目標專案路徑欄位，AI 每次呼叫下游工具時必須明確聲明所在專案，由工具結構層面強制執行，不依賴任何規則或記憶。
- **Gateway 啟動工作目錄感知**：Gateway 啟動時自動讀取 IDE 注入的環境變數，並支援 `--workspace=` 啟動參數，偵測結果自動套用為初始工作目錄設定。
- **工作目錄管理工具**：新增 `gateway__set_workspace` 與 `gateway__get_workspace` 兩個管理工具，支援手動宣告與查詢目前工作目錄，管理工具總數由 10 個增至 12 個。

---

## v0.4.0 — 2026-03-28

### 新增功能

- **IDE 工作目錄感知**：新增兩個管理工具，讓 AI 在對話開始時可明確宣告正在操作的目標專案目錄。後續所有需要存取專案路徑的工具（如程式碼品質掃描、網頁測試等）均可正確定位目標位置，不再以閘道器自身安裝目錄為基準。管理工具總數由 10 個增至 12 個，新增 4 個自動化測試案例，全站測試 67 個全數通過。

---

## v0.3.0 — 2026-03-28

### 安全修復

- **依賴漏洞清除**：修復兩個高嚴重度安全漏洞（picomatch ReDoS, CVSS 7.5；path-to-regexp ReDoS），升級底層工具鏈間接依賴版本。原始碼零修改，不影響生產執行環境。

### 框架與工具更新

- **Antigravity 框架升級**：同步最新版框架規則（核心指令規範）、工作流程（除錯、健檢）、技能（程式碼審計任務提示詞、CLI 委派 SOP），確保 AI 協作流程與主倉庫保持一致。
- **記憶管理卡夾系統整合**：安裝 Cartridge System MCP，Gateway 成功掃描並註冊 8 個記憶卡夾管理工具，可透過 AI 直接操作記憶卡夾的讀取、更新與索引管理功能。

---

## v0.2.0 — 2026-03-28

### 新增功能

- **參數型別容錯強轉**：Gateway 在轉發工具呼叫前，自動根據下游 MCP 的參數規格修正型別（字串→數字、字串→布林值等），避免不同 AI 模型或 IDE 因型別推斷差異導致工具呼叫失敗。

### 問題修復

- **CLI 安裝流程多行 JSON 輸入修復**：解決使用者在主控台貼入多行 JSON 設定檔時，因 readline 逐行消化導致 JSON 解析失敗的問題。改用事件監聽累積機制替代原有的逐行讀取策略。
- **新增 mcpServers/扁平 JSON 雙格式解析**：支援使用者直接貼入 Claude/Cline 標準格式或扁平格式的 MCP 設定，自動辨識並跳轉安裝流程。
